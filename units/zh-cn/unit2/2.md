# 直接偏好优化 (DPO)

直接偏好优化 (Direct Preference Optimization, DPO) 通过提供一种更简单、更稳定的替代方案来彻底改变偏好对齐，替代了人类反馈强化学习 (Reinforcement Learning from Human Feedback, RLHF)。DPO 不是训练单独的奖励模型和使用复杂的强化学习算法，而是使用人类偏好数据直接优化语言模型。

## 理解 DPO

![dpo diagram](https://github.com/huggingface/trl/assets/49240599/9150fac6-3d88-4ca2-8ec6-2a6f3473216d)

传统的 RLHF 方法需要多个组件和训练阶段。如图所示，它涉及：
- 训练奖励模型以根据偏好和拒绝的响应预测人类偏好。
- 使用 PPO 等强化学习算法针对奖励模型优化策略。

DPO 通过跳过奖励模型并使用二元交叉熵损失直接优化语言模型来极大地简化这个过程。
- 首先，训练 SFT 模型以遵循指令。
- 然后，训练 DPO 模型使用偏好数据本身直接优化语言模型。

> [!TIP]
> DPO 已被证明如此有效，以至于它被用于训练生产模型，如 Meta 的 Llama 系列和许多其他最先进的语言模型。

## DPO 如何工作

DPO 将偏好对齐重新构造为分类问题。给定一个提示和两个响应（一个偏好，一个拒绝），DPO 训练模型增加偏好响应的可能性，同时减少拒绝响应的可能性。

### 训练过程

DPO 过程需要监督微调 (SFT) 来使模型适应目标领域。这通过在标准指令遵循数据集上训练为偏好学习创建了基础。模型学习基本的任务完成，同时保持其通用能力。

接下来是偏好学习，在这里模型在输出对上训练——一个偏好的和一个非偏好的。偏好对帮助模型理解哪些响应更好地与人类价值观和期望对齐。

DPO 的核心创新在于其直接优化方法。DPO 不是训练单独的奖励模型，而是使用二元交叉熵损失根据偏好数据直接更新模型权重。这种简化的过程使训练更加稳定和高效，同时实现与传统 RLHF 可比或更好的结果。

### DPO 损失函数

DPO 的核心创新在于其损失函数，它使用偏好数据直接优化策略（语言模型）：

$$L_{DPO} = -\mathbb{E}_{(\pi,r)\sim D} \left[\log \sigma\left(\beta \log \frac{\pi_\theta(y_w|x)}{\pi_{ref}(y_w|x)} - \beta \log \frac{\pi_\theta(y_l|x)}{\pi_{ref}(y_l|x)}\right)\right]$$

其中：
- `π_θ` 是正在训练的模型
- `π_ref` 是参考模型（通常是 SFT 模型）
- `y_w` 是偏好（获胜）响应
- `y_l` 是拒绝（失败）响应
- `β` 是控制优化强度的温度参数
- `σ` 是 sigmoid 函数

## DPO 数据集格式

DPO 训练需要偏好数据集，其中每个示例包含：

| 字段 | 描述 | 示例 |
|-------|-------------|---------|
| `prompt` | 输入提示或问题 | "用简单的术语解释量子计算" |
| `chosen` | 偏好的响应 | "量子计算使用量子力学原理..." |
| `rejected` | 较不偏好的响应 | "量子计算非常复杂且难以理解..." |

数据集还可以包含额外的特征以增强训练质量。它可以包括为模型行为提供指令的系统提示。它还可以包含涉及带有偏好注释的复杂对话的多轮对话。最后，它可以包含提供额外上下文的元数据，如偏好强度或注释者一致性。

## 使用 TRL 实现

[TRL (Transformers Reinforcement Learning)](https://huggingface.co/docs/trl) 库使 DPO 实现变得简单：

```python
from trl import DPOConfig, DPOTrainer
from transformers import AutoModelForCausalLM, AutoTokenizer
from datasets import load_dataset

# 加载模型和分词器
model = AutoModelForCausalLM.from_pretrained("HuggingFaceTB/SmolLM3-3B")
tokenizer = AutoTokenizer.from_pretrained("HuggingFaceTB/SmolLM3-3B")

# 配置 DPO 训练
training_args = DPOConfig(
    beta=0.1,                    # 温度参数
    learning_rate=5e-7,          # 比 SFT 更低以保持稳定性
    max_prompt_length=512,       # 最大提示长度
    max_length=1024,             # 最大总长度
    per_device_train_batch_size=4,
    gradient_accumulation_steps=4,
    num_train_epochs=1,
)

# 初始化训练器
trainer = DPOTrainer(
    model=model,
    args=training_args,
    train_dataset=preference_dataset,
    processing_class=tokenizer,
)

# 训练模型
trainer.train()
```

## 期望的数据集类型

DPO 需要一个[偏好数据集](https://huggingface.co/docs/trl/en/dataset_formats#preference)。`DPOTrainer` 支持[对话式](https://huggingface.co/docs/trl/en/dataset_formats#conversational)和[标准](https://huggingface.co/docs/trl/en/dataset_formats#standard)数据集格式。当提供对话式数据集时，训练器将自动将聊天模板应用于数据集。

尽管 `DPOTrainer` 支持显式和隐式提示，但我们建议使用显式提示。如果提供隐式提示数据集，训练器将自动从 `"chosen"` 和 `"rejected"` 列中提取提示。有关更多信息，请参阅[偏好样式](dataset_formats#preference)部分。

尽管 `DPOTrainer` 支持显式和隐式提示，但我们建议使用显式提示。如果提供隐式提示数据集，训练器将自动从 `"chosen"` 和 `"rejected"` 列中提取提示。有关更多信息，请参阅[偏好样式](https://huggingface.co/docs/trl/en/dataset_formats#preference)部分。

| 参数 | 描述 | 建议 |
|-----------|-------------|-----------------|
| **Beta (β)** | 控制偏好优化的强度 | **范围**: 0.1 到 0.5<br>**较低值**: 更保守，接近参考模型<br>**较高值**: 更强的偏好对齐，过拟合风险 |
| **学习率** | DPO 训练的学习率 | **建议**: 比标准微调低得多 (5e-7 到 5e-6)<br>**原理**: 防止灾难性遗忘并保持稳定性<br>**调整**: 如果训练变得不稳定则进一步降低 |
| **数据集大小和质量** | 偏好数据集的要求 | **最小值**: 约 1,000 个高质量偏好对用于特定领域任务<br>**推荐**: 10,000+ 对用于稳健对齐<br>**质量优于数量**: 更好的做法是拥有更少的高质量对而不是许多低质量对 |

## 最佳实践

### 数据质量

数据质量对于成功的 DPO 训练至关重要。偏好数据集应包含涵盖所需行为的不同方面的多样化示例。清晰的注释指南确保偏好和拒绝响应的一致标记。你可以通过提高偏好数据集的质量来提高模型性能。例如，通过过滤更大的数据集以仅包含高质量示例，或与你的用例相关的示例。

在训练期间，仔细监控损失收敛并在保留数据上验证性能。beta 参数可能需要调整以平衡偏好学习与保持模型的通用能力。对不同提示进行定期评估有助于确保模型正在学习预期的偏好而不会过拟合。

将模型的输出与参考模型进行比较以验证偏好对齐的改进。在各种提示上进行测试，包括边缘情况，有助于确保在不同场景中稳健的偏好学习。

### 训练稳定性

在训练期间仔细监控损失收敛——DPO 损失应平稳下降，没有振荡或不稳定的行为。定期将你的模型的输出与参考模型进行比较，以确保你看到偏好对齐的有意义改进。使用梯度裁剪来防止训练不稳定，特别是在使用较高学习率或具有挑战性的数据集时。实施早停机制以在性能停滞或开始下降时停止训练，防止过拟合和浪费计算资源。

### 评估

在各种提示上评估模型的性能，包括边缘情况，以确保在不同场景中稳健的偏好学习。将模型的输出与参考模型进行比较以验证偏好对齐的改进。

### 避免常见陷阱

在实现 DPO 时，要注意对偏好的过拟合，这可能导致模型变得重复或失去通用能力。如果发生这种情况，降低 beta 参数，减少训练时间，或增加数据集多样性以保持更广泛的能力。相反，如果你注意到尽管训练但对齐几乎没有改进或没有改进，偏好信号可能不足——尝试增加 beta 参数，提高数据集质量，或延长训练时间。

另一个常见问题是分布偏移，即模型在训练领域上表现良好但难以泛化到新场景。为避免这种情况，确保你的偏好数据集全面覆盖目标用例，并包含代表真实世界应用的多样化示例。目标是实现稳健的偏好学习，在不同上下文中保持模型的实用性。
