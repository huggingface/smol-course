## 练习 1：探索 SmolLM3 的对话模板

**目标**：了解 SmolLM3 如何处理不同的对话格式和推理模式。

SmolLM3 是一个混合推理模型，可以遵循指令或生成在复杂问题上"推理"的词元。当有效进行后训练时，模型将对难题进行推理，并对简单问题生成直接响应。

### 环境设置

> [!WARNING]
> - 你需要至少 8GB VRAM 的 GPU 进行训练。CPU/MPS 可以运行格式化和数据集探索，但训练较大模型可能会失败。
> - 首次运行将下载几 GB 的模型权重；确保有 15GB+ 的可用磁盘空间和稳定的连接。
> - 如果你需要访问私有仓库，请通过 `login()` 向 Hugging Face Hub 进行身份验证。

让我们从设置环境开始。


### 探索对话模板格式

现在让我们探索对话模板格式。我们将创建不同类型的对话进行测试。

```python
# 创建不同类型的对话进行测试
conversations = {
    "simple_qa": [
        {"role": "user", "content": "What is machine learning?"},
    ],

    "with_system": [
        {"role": "system", "content": "You are a helpful AI assistant specialized in explaining technical concepts clearly."},
        {"role": "user", "content": "What is machine learning?"},
    ],

    "multi_turn": [
        {"role": "system", "content": "You are a math tutor."},
        {"role": "user", "content": "What is calculus?"},
        {"role": "assistant", "content": "Calculus is a branch of mathematics that deals with rates of change and accumulation of quantities."},
        {"role": "user", "content": "Can you give me a simple example?"},
    ],

    "reasoning_task": [
        {"role": "user", "content": "Solve step by step: If a train travels 120 miles in 2 hours, what is its average speed?"},
    ]
}

for conv_type, messages in conversations.items():
    print(f"--- {conv_type.upper()} ---")

    # 不带生成提示词格式化（用于完整对话）
    formatted_complete = instruct_tokenizer.apply_chat_template(
        messages,
        tokenize=False,
        add_generation_prompt=False
    )

    # 带生成提示词格式化（用于推理）
    formatted_prompt = instruct_tokenizer.apply_chat_template(
        messages,
        tokenize=False,
        add_generation_prompt=True
    )

    print("Complete conversation format:")
    print(formatted_complete)
    print("\nWith generation prompt:")
    print(formatted_prompt)
    print("\n" + "="*50 + "\n")
```

### 比较基础模型与指令模型的响应

在本节中，我们在基础模型和指令模型上运行相同的提示词，以观察格式差异以及对话模板如何影响生成质量和风格。

```python
# 在两个模型上测试相同的提示词
test_prompt = "Explain quantum computing in simple terms."

# 为基础模型准备提示词（无对话模板）
base_inputs = base_tokenizer(test_prompt, return_tensors="pt").to(device)

# 为指令模型准备提示词（带对话模板）
instruct_messages = [{"role": "user", "content": test_prompt}]
instruct_formatted = instruct_tokenizer.apply_chat_template(
    instruct_messages,
    tokenize=False,
    add_generation_prompt=True
)
instruct_inputs = instruct_tokenizer(instruct_formatted, return_tensors="pt").to(device)

# 生成响应
print("=== Model comparison ===\n")

print("🤖 BASE MODEL RESPONSE:")
with torch.no_grad():
    base_outputs = base_model.generate(
        **base_inputs,
        max_new_tokens=150,
        temperature=0.7,
        do_sample=True,
        pad_token_id=base_tokenizer.eos_token_id
    )
    base_response = base_tokenizer.decode(base_outputs[0], skip_special_tokens=True)
    print(base_response[len(test_prompt):])  # 仅显示生成的部分

print("\n" + "="*50)
print("Instruct model response:")
with torch.no_grad():
    instruct_outputs = instruct_model.generate(
        **instruct_inputs,
        max_new_tokens=150,
        temperature=0.7,
        do_sample=True,
        pad_token_id=instruct_tokenizer.eos_token_id
    )
    instruct_response = instruct_tokenizer.decode(instruct_outputs[0], skip_special_tokens=True)
    # 仅提取助手的响应
    assistant_start = instruct_response.find("<|im_start|>assistant\n") + len("<|im_start|>assistant\n")
    assistant_response = instruct_response[assistant_start:].split("<|im_end|>")[0]
    print(assistant_response)
```

如果我们深入研究下面的输出，我们可以看到基础模型和指令模型之间的差异。简而言之，基础模型继续字符串，而指令模型使用对话模板。例如，基础模型以 `" What are the differences between the classical bit and the quantum bit?"` 开头，而指令模型通过回答问题开始：`"Quantum computing is a type of computing that uses quantum bits"`。

### 处理不同的数据集类型

SmolTalk2 数据集是为方便而编译在一起的开源数据集集合。它包含有用的后训练用例的混合，如工具使用、长上下文等。它们都采用对话格式，易于用于训练。但是，并非所有数据集都以一致的格式共享，因此我们经常需要将它们处理成统一的对话 `messages` 布局。

对于本练习，我们将多个数据集格式标准化为统一的对话 `messages` 布局。我们为 QA 和指令数据集定义轻量级处理器，并使用 GSM8K 演练一个具体示例。

```python
# 处理不同数据集格式的函数
def process_qa_dataset(examples, question_col, answer_col):
    """将 Q&A 数据集处理成对话格式"""
    processed = []

    for question, answer in zip(examples[question_col], examples[answer_col]):
        messages = [
            {"role": "user", "content": question},
            {"role": "assistant", "content": answer}
        ]
        processed.append(messages)

    return {"messages": processed}

def process_instruction_dataset(examples):
    """处理指令遵循数据集"""
    processed = []

    for instruction, response in zip(examples["instruction"], examples["response"]):
        messages = [
            {"role": "user", "content": instruction},
            {"role": "assistant", "content": response}
        ]
        processed.append(messages)

    return {"messages": processed}

# 使用 GSM8K 数据集的示例
print("=== PROCESSING GSM8K DATASET ===\n")
gsm8k = load_dataset("openai/gsm8k", "main", split="train[:100]")
print(f"Original dataset: {gsm8k}")

# 处理成对话格式
processed_gsm8k = gsm8k.map(
    lambda x: process_qa_dataset(x, "question", "answer"),
    batched=True,
    remove_columns=gsm8k.column_names
)

print(f"Processed dataset: {processed_gsm8k}")
print(f"\nExample conversation:")
print(processed_gsm8k[0]["messages"])
```

**目标**：使用 Python API 在 SmolTalk2 数据集上微调 SmolLM3。

```python
from datasets import load_dataset
```

### 步骤 2：配置训练

```python
from trl import SFTConfig

training_config = SFTConfig(
    output_dir="./smollm3-sft-output",

    # 训练超参数
    per_device_train_batch_size=2,
    gradient_accumulation_steps=2,
    learning_rate=5e-5,
    num_train_epochs=1,
    max_steps=100,  # 限制步数以进行快速测试

    # 优化
    warmup_steps=10,
    logging_steps=10,
    save_steps=50,

    # 内存和性能
    bf16=True,
    remove_unused_columns=False,

    # Hub 集成
    push_to_hub=False,  # 如果要上传，设置为 True
    # hub_model_id="your-username/smollm3-sft",
)

print("Training configuration set!")
print(f"Effective batch size: {training_config.per_device_train_batch_size * training_config.gradient_accumulation_steps}")
```

### 可选：使用 LoRA/PEFT 训练（节省内存）

如果你的 GPU 内存有限或希望更快迭代，请通过 PEFT 使用 LoRA。这仅训练小型适配器权重，同时保持基础模型冻结，然后你可以继续使用适配器或稍后合并它们以进行部署。

```python
# 使用 PEFT 的 LoRA 配置
from peft import LoraConfig

peft_config = LoraConfig(
    r=8,
    lora_alpha=16,
    lora_dropout=0.05,
    bias="none",
    task_type="CAUSAL_LM",
)

# 创建启用 LoRA 的 SFTTrainer
from trl import SFTTrainer

lora_trainer = SFTTrainer(
    model=model,
    train_dataset=formatted_dataset,
    args=training_config,
    peft_config=peft_config,  # << 启用 LoRA
)

print("Starting LoRA training…")
lora_trainer.train()
```

## 练习 4：使用 TRL CLI 的生产工作流

在之前的练习中，我们深入研究了使用 TRL 的 Python API 进行微调，并探索了我们正在使用和生成的数据。在本练习中，我们将探索使用 TRL CLI 来微调模型。这将是在生产中微调模型的最常见方式。

我们可以在 TRL CLI 中定义一个命令来微调模型。我们将能够使用 `trl sft` 命令运行它。CLI 命令和 Python API 共享相同的配置选项。

我们预处理了 SmolTalk2 的 `smoltalk_everyday_convs_reasoning_Qwen3_32B_think` 子集，以便在使用 TRL CLI 时更容易使用它。

```bash
# 使用 TRL CLI 微调 SmolLM3
trl sft \
    --model_name_or_path HuggingFaceTB/SmolLM3-3B-Base \
    --dataset_name HuggingFaceTB/smoltalk2_everyday_convs_think \
    --output_dir ./smollm3-sft-cli \
    --per_device_train_batch_size 4 \
    --gradient_accumulation_steps 2 \
    --learning_rate 5e-5 \
    --num_train_epochs 1 \
    --max_length 2048 \
    --logging_steps 10 \
    --save_steps 500 \
    --warmup_steps 100 \
    --bf16 True \
    --push_to_hub \
    --hub_model_id your-username/smollm3-sft-cli
```

为了方便和可重现性，我们还可以创建一个配置文件来微调模型。例如，我们可以创建一个名为 `sft_config.yaml` 的文件，并在其中放入以下内容：

```yaml
# 模型和数据集
model_name_or_path: HuggingFaceTB/SmolLM3-3B-Base
dataset_name: HuggingFaceTB/smoltalk2_everyday_convs_think
output_dir: ./smollm3-advanced-sft

# 训练超参数
per_device_train_batch_size: 2
gradient_accumulation_steps: 4
learning_rate: 3e-5
num_train_epochs: 2
max_length: 4096

# 优化
warmup_steps: 200
weight_decay: 0.01
optim: adamw_torch
lr_scheduler_type: cosine

# 内存和性能
bf16: true
dataloader_num_workers: 4
group_by_length: true
remove_unused_columns: false

# 日志记录和评估
logging_steps: 25
eval_steps: 250
save_steps: 500
eval_strategy: steps
load_best_model_at_end: true
metric_for_best_model: eval_loss

# Hub 集成
push_to_hub: true
hub_model_id: your-username/smollm3-advanced
hub_strategy: every_save
```

然后我们可以将此文件提交到仓库并使用 Git 跟踪它。

```bash
# 使用配置文件运行训练
trl sft --config sft_config.yaml
```

### 故障排除

**如果出现 GPU 内存不足错误：**
- 将 `per_device_train_batch_size` 减少到 1
- 将 `max_length` 减少到 1024 或 512
- 使用 `torch.cuda.empty_cache()` 清除 GPU 内存

**如果模型加载失败：**
- 检查你的互联网连接
- 尝试使用 `device_map="cpu"` 进行 CPU 加载
- 使用较小的模型如 `HuggingFaceTB/SmolLM3-1.7B` 进行测试

**如果训练失败：**
- 确保你的数据集格式正确
- 检查所有示例的长度合理（不要太长）
- 监控训练损失 - 它应该稳定下降
