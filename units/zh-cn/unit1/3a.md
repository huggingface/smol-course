# LoRA 和 PEFT：高效微调

参数高效微调 (Parameter-Efficient Fine-Tuning, PEFT) 让你通过训练少量额外参数来适应大型模型，同时保持基础模型冻结。最广泛使用的 PEFT 方法是 LoRA（低秩适应，Low-Rank Adaptation），它将可训练的低秩更新注入线性层。这通常可以减少约 90% 的可训练参数，同时保持性能。

## 何时使用 PEFT

- 你的计算或内存预算有限
- 你需要快速将基础模型适应到多个任务/领域
- 你希望快速迭代和小型工件（适配器权重通常只有几 MB）

## 理解 LoRA

LoRA 已成为最广泛采用的 PEFT 方法。它通过向注意力权重添加小的秩分解矩阵来工作，通常将可训练参数减少约 90%。

LoRA（低秩适应）是一种参数高效的微调技术，它冻结预训练模型权重并将可训练的秩分解矩阵注入模型的层中。LoRA 不是在微调期间训练所有模型参数，而是通过低秩分解将权重更新分解为较小的矩阵，从而显著减少可训练参数的数量，同时保持模型性能。例如，当应用于 GPT-3 175B 时，LoRA 与完全微调相比，将可训练参数减少了 10,000 倍，GPU 内存需求减少了 3 倍。你可以在 [LoRA 论文](https://huggingface.co/papers/2106.09685)中阅读更多关于 LoRA 的信息。

LoRA 通过向 Transformer 层添加成对的秩分解矩阵来工作，通常关注注意力权重。在推理期间，这些适配器权重可以与基础模型合并，不会产生额外的延迟开销。LoRA 特别适用于将大型语言模型适应到特定任务或领域，同时保持资源需求可管理。

## 加载 LoRA 适配器

适配器可以使用 `load_adapter()` 加载到预训练模型上，这对于尝试不同的适配器（其权重未合并）非常有用。使用 `set_adapter()` 函数设置活动适配器权重。要返回基础模型，你可以使用 `unload()` 卸载所有 LoRA 模块。这使得在不同的特定任务权重之间切换变得容易。

```python
from transformers import AutoModelForCausalLM
from peft import PeftModel

base_model = AutoModelForCausalLM.from_pretrained("<base_model_name>")
peft_model_id = "<peft_adapter_id>"
model = PeftModel.from_pretrained(base_model, peft_model_id)
```

## 合并 LoRA 适配器

使用 LoRA 训练后，你可能希望将适配器权重合并回基础模型，以便更轻松地部署。这会创建一个具有组合权重的单一模型，无需在推理期间单独加载适配器。

合并过程需要注意内存管理和精度。由于你需要同时加载基础模型和适配器权重，请确保有足够的 GPU/CPU 内存可用。在 `transformers` 中使用 `device_map="auto"` 将有助于自动内存管理。在整个过程中保持一致的精度（例如，float16），匹配训练期间使用的精度，并以相同格式保存合并模型以进行部署。在部署之前，始终通过将合并模型的输出和性能指标与基于适配器的版本进行比较来验证合并模型。

适配器也便于在不同任务或领域之间切换。你可以分别加载基础模型和适配器权重。这允许在不同的特定任务权重之间快速切换。

> [!TIP]
> 在实现 PEFT 方法时，从小的秩值（4-8）开始用于 LoRA 并监控训练损失。使用验证集防止过拟合，并在可能的情况下将结果与完全微调基线进行比较。不同方法的有效性可能因任务而异，因此实验是关键。

## OLoRA

[OLoRA](https://huggingface.co/papers/2406.01775) 利用 QR 分解来初始化 LoRA 适配器。OLoRA 通过其 QR 分解的因子来平移模型的基础权重，即在对其进行任何训练之前对权重进行变异。这种方法显著提高了稳定性，加速了收敛速度，并最终实现了卓越的性能。

## 将 TRL 与 PEFT 结合使用

PEFT 方法可以与 TRL（Transformers Reinforcement Learning）结合使用以实现高效微调。这种集成对于 RLHF（人类反馈强化学习）特别有用，因为它减少了内存需求。

```python
from peft import LoraConfig
from trl import SFTTrainer

# 使用 PEFT 配置加载模型
lora_config = LoraConfig(
    r=16,
    lora_alpha=32,
    lora_dropout=0.05,
    bias="none",
    task_type="CAUSAL_LM"
)

trainer = SFTTrainer(
    model="your-model-name",
    train_dataset=dataset["train"]
    peft_config=lora_config
)
```

## 基本合并实现

训练 LoRA 适配器后，你可以将适配器权重合并回基础模型。以下是操作方法：

```python
import torch
from transformers import AutoModelForCausalLM
from peft import PeftModel

# 1. 加载基础模型
base_model = AutoModelForCausalLM.from_pretrained(
    "base_model_name",
    dtype=torch.bfloat16,
    device_map="auto"
)

# 2. 使用适配器加载 PEFT 模型
peft_model = PeftModel.from_pretrained(
    base_model,
    "path/to/adapter",
    dtype=torch.bfloat16
)

# 3. 将适配器权重与基础模型合并
try:
    merged_model = peft_model.merge_and_unload()
except RuntimeError as e:
    print(f"Merging failed: {e}")
    # 实现回退策略或内存优化

# 4. 保存合并的模型
merged_model.save_pretrained("path/to/save/merged_model")
```

如果你在保存的模型中遇到大小差异，请确保也保存分词器：

```python
# 同时保存模型和分词器
tokenizer = AutoTokenizer.from_pretrained("base_model_name")
merged_model.save_pretrained("path/to/save/merged_model")
tokenizer.save_pretrained("path/to/save/merged_model")
```

## 使用 TRL + LoRA 快速入门

TRL `SFTTrainer` 与 PEFT 原生集成。定义一个 `LoraConfig`，将其传递给训练器，并仅训练适配器权重。

```python
from peft import LoraConfig
from trl import SFTTrainer, SFTConfig

# 1) 配置 LoRA
peft_config = LoraConfig(
    r=8,
    lora_alpha=16,
    lora_dropout=0.05,
    bias="none",
    task_type="CAUSAL_LM",
)

# 2) 创建训练器（示例）
trainer = SFTTrainer(
    model=model,
    args=SFTConfig(output_dir="lora-adapter", num_train_epochs=1, per_device_train_batch_size=2, packing=True),
    train_dataset=dataset["train"],
    peft_config=peft_config,
)
trainer.train()
```

训练后，你可以：

- 在推理时加载适配器与基础模型一起，或
- 将适配器合并到基础模型中以简化部署。
