# 使用 Hugging Face Jobs 训练

**Hugging Face Jobs** 提供完全托管的云基础设施，用于训练模型，无需设置 GPU、管理依赖项或在本地配置环境的麻烦。这对于 SFT 训练特别有价值，因为它可能需要大量资源和时间。

## 为什么使用 Jobs 进行 SFT 训练？

- **可扩展的基础设施**：无需硬件投资即可访问高端 GPU（A100、L4 等）
- **零设置**：无需管理 CUDA 驱动程序、Docker 容器或环境配置
- **成本效益**：仅为使用的计算时间付费，完成后自动关闭
- **集成工作流**：与 Hugging Face Hub 无缝集成，用于模型存储和共享
- **监控**：通过 Hub 界面内置日志记录和进度跟踪

## 要求

要使用 Hugging Face Jobs，你需要：
- **Pro、Team 或 Enterprise** Hugging Face 计划，你可以在[这里](https://huggingface.co/pricing)获取
- 通过 `hf auth login` 进行身份验证

## 使用 Jobs 运行 SFT：两种方法

使用 HF jobs 运行 TRL 的最佳方法是使用内置脚本。它们利用 `uv` 来管理依赖项，并使用 `hf jobs` 来运行训练作业。

本指南将引导你使用 TRL 的内置脚本通过 Hugging Face Jobs 训练模型。如果你想使用自定义脚本，你可以实现 `uv` 依赖项并使用 `hf jobs run` 运行脚本。

<details>

<summary>创建带有内联依赖项的自定义训练脚本</summary>

```python
# sft_training.py
# /// script
# dependencies = [
#     "trl[sft]>=0.7.0",
#     "transformers>=4.36.0",
#     "datasets>=2.14.0",
#     "accelerate>=0.24.0",
#     "peft>=0.7.0"
# ]
# ///

from trl import SFTTrainer, SFTConfig
from transformers import AutoModelForCausalLM, AutoTokenizer
from datasets import load_dataset

# 加载模型和分词器
model = AutoModelForCausalLM.from_pretrained("HuggingFaceTB/SmolLM3-3B-Base")
tokenizer = AutoTokenizer.from_pretrained("HuggingFaceTB/SmolLM3-3B-Base")

# 加载数据集
dataset = load_dataset("HuggingFaceTB/smoltalk2", "SFT")

# 配置训练
config = SFTConfig(
    output_dir="./smollm3-jobs-sft",
    per_device_train_batch_size=4,
    learning_rate=5e-5,
    max_steps=1000,
    logging_steps=50,
    save_steps=200,
    push_to_hub=True,
    hub_model_id="your-username/smollm3-jobs-sft"
)

# 训练
trainer = SFTTrainer(
    model=model,
    train_dataset=dataset["smoltalk_everyday_convs_reasoning_Qwen3_32B_think"],
    args=config,
)
trainer.train()
```

然后使用 Jobs CLI 运行：

```bash
# 在 Jobs 上运行 UV 脚本
hf jobs uv run \
    --flavor a10g-large \
    --timeout 2h \
    --secrets HF_TOKEN \
    sft_training.py
```

</details>

## SFT 的硬件选择

根据你的模型大小和训练要求选择合适的硬件类型：

**对于 SmolLM3-3B（推荐）**：
- `a10g-large`：24GB GPU 内存，对于大多数 SFT 任务具有成本效益
- `a100-large`：40GB GPU 内存，使用更大批次大小实现最快训练
- `l4x1`：24GB GPU 内存，用于分布式训练的多 GPU 设置

**对于更大的模型（7B+）**：
- `a100-large`：7B+ 模型所需
- `l4x4`：用于分布式训练的多 GPU 设置

**预算选项**：
- `t4-small`：16GB GPU 内存，较慢但经济实惠，适合实验
- `l4x1`：24GB GPU 内存，成本和性能的良好平衡

> [!TIP]
> 有关不同硬件类型的详细比较，你可以查看[定价页面](https://huggingface.co/pricing)。

## 高级 Jobs 配置

```bash
# 直接使用 TRL 维护的 SFT 脚本
hf jobs uv run \
    --flavor a10g-large \
    --timeout 2h \
    --secrets HF_TOKEN \
    "https://raw.githubusercontent.com/huggingface/trl/main/trl/scripts/sft.py" \
    --model_name_or_path HuggingFaceTB/SmolLM3-3B-Base \
    --dataset_name HuggingFaceTB/smoltalk2_everyday_convs_think \
    --learning_rate 5e-5 \
    --per_device_train_batch_size 4 \
    --max_steps 1000 \
    --output_dir smollm3-sft-jobs \
    --push_to_hub \
    --hub_model_id your-username/smollm3-sft \
    --report_to trackio
```

**环境变量和密钥**：

如果你正在使用自定义脚本，你可以使用 `--secrets` 标志传入环境变量。

```bash
hf jobs uv run \
    --flavor a10g-large \
    --timeout 3h \
    --secrets HF_TOKEN=your_token \
    --secrets WANDB_API_KEY=your_wandb_key \
    --env WANDB_PROJECT=smollm3-sft \
    --env CUDA_VISIBLE_DEVICES=0 \
    my_sft_training.py
```

## 监控你的训练作业

要检查你的训练作业，你可以使用 `hf jobs` 命令或在 Hub 上访问[作业设置](https://huggingface.co/settings/jobs)。

**检查作业状态**：

```bash
# 列出所有作业
hf jobs ps -a

# 获取详细的作业信息
hf jobs inspect <job_id>

# 实时流式传输作业日志
hf jobs logs <job_id> --follow

# 如果需要，取消正在运行的作业
hf jobs cancel <job_id>
```

## Jobs 上的 LoRA/PEFT（可选）

通过传递 PEFT 标志，在使用 TRL 维护的 SFT 脚本时启用 LoRA。有关权威标志和默认值，请参见脚本：`https://github.com/huggingface/trl/blob/main/trl/scripts/sft.py`。

```bash
hf jobs uv run \
  --flavor a10g-large \
  --timeout 2h \
  --secrets HF_TOKEN \
  "https://raw.githubusercontent.com/huggingface/trl/main/trl/scripts/sft.py" \
  --model_name_or_path HuggingFaceTB/SmolLM3-3B-Base \
  --dataset_name HuggingFaceTB/smoltalk2_everyday_convs_think \
  --output_dir smollm3-lora-sft-jobs \
  --per_device_train_batch_size 4 \
  --learning_rate 5e-5 \
  --max_steps 1000 \
  --report_to trackio \
  --push_to_hub \
  --hub_model_id your-username/smollm3-lora-sft \
  --use_peft \
  --lora_r 8 \
  --lora_alpha 16 \
  --lora_dropout 0.05 \
  --lora_target_modules all-linear
```

注意事项：
- 在运行之前确认 TRL SFT 脚本中的标志名称：`https://github.com/huggingface/trl/blob/main/trl/scripts/sft.py`。
- LoRA 训练小型适配器，你可以将其保持独立或稍后合并以进行部署。

## 成本估算

SmolLM3-3B SFT 训练（1000 步）的大致成本：
- l4x1：约 $3-4/小时（24GB GPU 内存）
- a10g-large：约 $4-6/小时（24GB GPU 内存）
- a100-large：约 $8-12/小时（40GB GPU 内存）

训练通常需要 30-90 分钟来完成 1000 步，具体取决于硬件和配置，与本地 GPU 租赁或云实例相比，Jobs 具有成本效益。

**节省成本的提示**：
- 使用较小的批次大小和梯度累积以适应更便宜的 GPU
- 从较短的训练运行（500 步）开始以验证你的设置
- 使用 `l4x1` 进行初始实验，然后扩展到更快的 GPU 用于生产
- 设置适当的超时以避免意外费用

## 常见问题故障排除

**内存不足错误**：
- 减少 `per_device_train_batch_size`
- 启用梯度检查点
- 使用较小的 `max_length`

**超时问题**：
- 增加超时参数
- 减少训练步数或使用更强大的硬件
- 优化数据加载和预处理

**身份验证错误**：
- 确保 HF_TOKEN 正确设置为密钥
- 验证你的 Hugging Face 账户具有所需计划
- 检查令牌权限以进行模型上传
