# 使用预训练的 VLM

视觉语言模型 (VLM) **同时处理图像和文本**，实现高级任务，如生成描述、回答视觉问题或跨模态推理。在本节中，我们专注于 **VLM 如何工作以及如何实际使用它们**。

## 架构概述

![VLM Architecture](https://huggingface.co/datasets/smol-course/images/resolve/main/VLM_Architecture.png)

VLM 结合图像处理和文本生成组件以实现统一的多模态理解。主要元素是：

- **图像/视觉编码器**：将图像转换为紧凑的数值表示。示例：CLIP、SigLIP。
- **嵌入投影器**：将图像特征与文本嵌入对齐（通常是为多模态任务微调的小型 MLP 或线性层）。
- **多模态投影器/融合模块**：融合并增强视觉和文本表示之间的连接。这一步超越了对齐，实现了丰富的跨模态交互。
- **文本解码器**：从融合的多模态表示生成文本（或其他输出）。

大多数 VLM 利用**预训练的图像编码器和文本解码器**，然后在配对的图像-文本数据集上进行微调，以实现高效训练和泛化。

## 实际用途

VLM 可以应用于以下任务：

- **图像描述**：为图像生成描述
- **视觉问答 (VQA)**：回答关于图像的问题
- **跨模态检索**：将图像与文本匹配，反之亦然
- **创意应用**：设计、艺术生成、多媒体内容

高质量的配对数据集是关键，🤗 transformers 提供预训练模型和简化的微调工作流。

![VLM Usage](https://huggingface.co/datasets/smol-course/images/resolve/main/VLM_Usage.png)

### 聊天格式

许多 VLM 支持**类似聊天的交互**，消息结构如下：

1. **系统消息**：设置上下文：`"你是一个分析视觉数据的助手。"`
2. **用户查询**：结合文本和图像。
3. **助手响应**：基于多模态分析生成的文本。

示例：

```json
[
  {
    "role": "system",
    "content": [{"type": "text", "text": "你是一个专门从事图表分析的 VLM。"}]
  },
  {
    "role": "user",
    "content": [
      {"type": "image", "image": "<image_data>"},
      {"type": "text", "text": "此图表中的最高值是多少？"}
    ]
  },
  {
    "role": "assistant",
    "content": [{"type": "text", "text": "42"}]
  }
]
```

VLM 还可以通过同一聊天模板传递图像序列，处理**多个图像或视频帧**作为输入。

## 通过 pipeline 使用 VLM

正如我们在[第 1 单元](../unit1/2)中看到的，使用 VLM 的最简单方法是通过 🤗 `pipeline` 抽象：

```python
from transformers import pipeline

# 使用 VLM 初始化 pipeline
pipe = pipeline("image-text-to-text", "HuggingFaceTB/SmolVLM2-2.2B-Instruct", device_map="auto")

# 使用图像定义你的对话
messages = [
     {
         "role": "user",
         "content": [
             {
                 "type": "image",
                 "image": "https://huggingface.co/datasets/huggingface/documentation-images/resolve/main/bee.jpg",
             },
             {"type": "text", "text": "描述这张图片。"},
         ],
     }
 ]

outputs = pipe(text=messages, max_new_tokens=60, return_full_text=False)

# 生成响应 - pipeline 自动处理多模态输入
response = pipe(messages, max_new_tokens=128, temperature=0.7)

print(response[0]['generated_text'][-1]['content'])  # 打印模型的描述
```

## 通过 Transformers 使用 VLM（完全控制）

对于高级使用，你可以直接通过 🤗 Transformers 访问 VLM，为你提供**对每个组件的完全控制**。
为了减少内存使用并加速推理，我们可以使用 `bitsandbytes` 应用 **4 位量化**。

与标准 LLM 使用不同，VLM 需要**处理器**而不仅仅是分词器。处理器处理**文本分词**和**图像预处理**，简化多模态输入的工作流。

```python
import torch
from transformers import AutoProcessor, AutoModelForImageTextToText, BitsAndBytesConfig
from transformers.image_utils import load_image

device = "cuda" if torch.cuda.is_available() else "cpu"

# 量化以提高效率
quant_config = BitsAndBytesConfig(load_in_4bit=True)
model_name = "HuggingFaceTB/SmolVLM2-2.2B-Instruct"
model = AutoModelForImageTextToText.from_pretrained(model_name, quantization_config=quant_config).to(device)
processor = AutoProcessor.from_pretrained(model_name)
```

### 示例：描述图像

我们可以使用**聊天模板**来描述图像。每个图像在消息中表示为 `{"type": "image"}`，实际图像数据通过 `images` 参数传递给处理器。处理器无缝处理文本和视觉输入。


```python
# 加载图像
image_url = "https://huggingface.co/datasets/huggingface/documentation-images/resolve/main/bee.jpg"
image = load_image(image_url)

# 创建输入消息
messages = [
    {
        "role": "user",
        "content": [
            {"type": "image"},
            {"type": "text", "text": "你能描述这张图片吗？"}
        ]
    },
]

# 准备输入
prompt = processor.apply_chat_template(messages, add_generation_prompt=True)
inputs = processor(text=prompt, images=[image], return_tensors="pt")
inputs = inputs.to(device)

# 生成输出
generated_ids = model.generate(**inputs, max_new_tokens=500)
generated_texts = processor.batch_decode(
    generated_ids,
    skip_special_tokens=True,
)[0]

# 仅提取助手响应
assistant_response = generated_texts.split("Assistant:")[-1].strip()

print(assistant_response)
```

<details>
<summary>输出</summary>

```text
The image is of a bee on a flower.
```

</details>

处理器结合文本和图像输入，因此模型可以生成连贯的多模态输出。

> [!TIP]
> 类似的模板可以处理多个图像、OCR 任务，甚至视频帧，使 VLM 非常通用。
