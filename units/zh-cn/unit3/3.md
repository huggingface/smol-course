# 微调 VLM

在第 1 单元中，我们探索了 [LLM 的监督微调](../unit1/3)，包括使用 TRL 的[高效策略](../unit1/3a)。在本节中，我们将这些技术适应于**视觉语言模型 (VLM)**，重点关注效率和特定任务性能。


## 关键效率技术

在微调 VLM 时，内存和计算很快就会成为瓶颈。以下是主要策略：

### 量化

量化降低模型权重和激活的精度，减少内存使用并加速计算。
- **bfloat16 / float16** 将内存需求减半，同时保持精度。
- **8 位 / 4 位量化** 进一步减少内存，性能权衡很小。
> ⚠️ 对于 VLM 尤其相关，因为图像特征会增加内存需求。

### PEFT 和 LoRA

低秩适应 (Low-Rank Adaptation, LoRA) 冻结基础模型权重并训练**紧凑的秩分解矩阵**，大幅减少可训练参数的数量。当与 **PEFT** 结合使用时，微调需要**数百万个可训练参数而不是数十亿个**，使大型 VLM 在有限硬件上变得可访问。

### 批次大小优化

内存高效训练可以通过以下方式实现：
- **梯度累积**：在多个步骤中保持有效批次大小。
- **梯度检查点**：重新计算中间激活以节省内存。
- 从大批次开始，如果发生 OOM 错误则减少，并与 LoRA/量化结合以获得最佳结果。


## 监督微调 (SFT)

SFT 使用标记数据集（图像-文本对）将预训练的 VLM 适应到**特定任务**。示例包括：
- 视觉问答 (VQA)
- 图像描述
- 图表或图解释

### 何时使用 SFT
- 在基础模型难以处理的领域中专门化 VLM。
- 学习特定领域的词汇或视觉模式。

**局限性**

- 需要**高质量的标记数据集**。
- 可能**计算密集**。
- 如果微调过于狭窄，存在**过拟合**风险。

### 使用示例

[`SFTTrainer`](https://huggingface.co/docs/trl/en/sft_trainer) 直接支持训练 VLM。
你的数据集应包括一个额外的 `images` 列，其中包含视觉输入。有关详细信息，请参阅[数据集格式文档](https://huggingface.co/docs/trl/dataset_formats#vision-dataset)。


```python
from trl import SFTTrainer

training_args = SFTTrainer(
   output_dir="./fine_tuned_model",
   per_device_train_batch_size=4,
   num_train_epochs=3,
   learning_rate=5e-5,
   save_steps=1000,
   bf16=True,
   gradient_checkpointing=True,
   gradient_accumulation_steps=16,
   logging_steps=50
)
```

⚠️ **重要**：在 [`SFTConfig`](https://huggingface.co/docs/trl/en/sft_trainer#trl.SFTConfig) 中设置 `max_length=None`。
否则，截断可能会在训练期间删除图像 token。

```python
SFTConfig(max_length=None, ...)
```

### 实践步骤

1. **数据准备**
   - 使用图像-文本对，例如 `HuggingFaceM4/ChartQA`。

2. **模型设置**
   - 加载预训练的 VLM，例如 `HuggingFaceTB/SmolVLM2-2.2B-Instruct`。
   - 初始化处理器以准备文本和图像输入。

3. **微调过程**
   - 将数据格式化为**类似聊天的消息**（`system`、`user`、`assistant`）。
   - 配置优化器、批次大小和梯度累积。
   - 应用**量化**和 **LoRA** 以实现内存高效训练。

## 偏好优化 (DPO)

**直接偏好优化 (DPO)** 将 VLM 与**人类偏好**对齐，而不是严格遵循指令。
- 适用于创意任务、主观判断或多选答案。
- 模型学习选择**更符合人类的响应**，即使它不是严格"正确"的。

**局限性**

- 需要**高质量的偏好标记数据集**。
- 训练涉及**成对偏好采样**和仔细的资源管理。

### 使用示例

- **数据集**：每个示例包含一个提示（图像 + 问题）和**两个候选响应**：

```text
Question: How many families?
Rejected: The image does not provide information about families.
Chosen: The image shows a Union Organization table setup with 18,000 families.
```

- **模型设置**：加载预训练的 VLM，与 TRL DPO 集成，并准备处理器。
- **训练流程**：
  - 将数据集格式化为类似聊天的消息。
  - 应用**基于偏好的损失函数**。
  - 使用梯度累积、检查点、LoRA 和量化以提高效率。

### SFT 与 DPO 比较

| 特性 | SFT | DPO |
|---------|-----|-----|
| 输入   | 标记的图像-文本 | 图像-文本 + 偏好排序输出 |
| 损失    | 标准监督 | 基于偏好 |
| 目标    | 特定任务适应 | 人类对齐输出 |
| 用例| 领域专业化 | 创意、主观或多选任务 |

### 实用技巧

- 从小处开始：在完全训练之前使用**数据集子集**进行测试。
- 使用**梯度检查点 + LoRA + 量化**来减少内存使用。
- 监控**检查点频率**以平衡存储和安全。
- 在小集合上验证以**避免过拟合**。

## 在 `hf jobs` 中使用 TRL 微调 VLM

正如在[早期单元](unit1/5)中介绍的，[Hugging Face Jobs](https://huggingface.co/docs/huggingface_hub/guides/jobs) 使微调视觉语言模型 (VLM) 变得简单。你可以直接在 Hugging Face 基础设施上运行**监督微调 (SFT)** 或**直接偏好优化 (DPO)**，设置最少，调整我们之前讨论的训练参数。

### 快速示例

```bash
hf jobs uv run \
   --flavor a100-large \
   --secrets HF_TOKEN \
   --timeout 2h \
   "https://raw.githubusercontent.com/huggingface/trl/main/trl/scripts/sft.py" \
   --model_name_or_path HuggingFaceTB/SmolVLM2-2.2B-Instruct \
   --dataset_name HuggingFaceM4/ChartQA \
   --report_to trackio
```

- `--flavor a100-large`：用于训练的 GPU 类型。
- `--secrets HF_TOKEN`：你的 Hugging Face token。

脚本自动处理处理器设置、数据格式化和模型训练。作业完成后，你的微调 VLM 就可以下载并在下游任务中使用了。

> [!TIP]
> 对于大型 VLM 的内存高效微调，考虑组合使用 **LoRA 适配器**、**梯度累积**和**量化**等技术。这些策略有助于减少内存使用，同时保持性能。
